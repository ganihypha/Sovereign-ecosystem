## VII. Session Types

Sovereign supports different session types:

### A. Doc Integration Session
- Focus: uploaded file intake + doc enhancement + merge/sync decisions
- Code execution: secondary or out of scope
- Output: repo-ready documentation

### B. Scoped Implementation Session
- Focus: specific feature/fix implementation
- Scope: tightly defined, with clear boundaries
- Output: verified implementation + updated docs

### C. Validation Session
- Focus: verify existing work + test coverage
- Scope: confirmation only, no new features
- Output: verification evidence + proof tracker update

### D. Gated Session
- Focus: validation THEN scoped execution
- Gate: must pass validation before proceeding
- Output: validated system + scoped implementation
