# STACK-07 — L3 Return / Proof Template

**Canonical ID:** STACK-07  
**Version:** v1.0  
**Status:** Canon  
**Layer Owner:** L3→L2 bridge  
**Primary Purpose:** Provide the official canon-grade structure L3 uses to return execution results, proof, risk, and unresolved items to L2.  
**Depends On:** STACK-00, STACK-06  
**Used By:** L3, L2, next L1 cycle  
**Drafting Mode:** canon-grade  
**Change Control:** Execution return / proof discipline change. Revisions affect auditability, completion taxonomy, and L2 verification governance.  
**Non-Negotiables:** No strong completion claim without proof. Executed, tested, completed, ready-for-review, and verified remain distinct. Blockers and unresolved items must be surfaced honestly.  
**Last Updated:** 2026-04-08

## 1. Purpose
This document defines the official return structure L3 uses after execution so L2 can review results without ambiguity, false completion, or proof-light claims.

## 3. Canon Rules
- No proof, no strong claim.  
- Executed ≠ Verified.  
- Partial is valid and must be stated honestly.  
- Blockers must be narratable.  
- L3 must disclose what it did not touch.
