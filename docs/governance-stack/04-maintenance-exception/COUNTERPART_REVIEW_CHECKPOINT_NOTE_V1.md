**Scope:** Template untuk evaluasi berkala status, fit, rhythm, dan privilege counterpart

## Instruksi Penggunaan

Checkpoint Note diisi secara berkala oleh Founder untuk mengevaluasi apakah level counterpart saat ini masih sesuai, apakah privilege perlu disesuaikan, dan apakah ada risiko yang perlu ditangani. Frekuensi checkpoint ditentukan oleh Founder — disarankan minimal setiap 30-90 hari setelah aktivasi.

## Catatan Penting

1. Checkpoint Note bersifat **confidential** dan hanya boleh diakses Founder.
2. Hasil checkpoint yang mengubah level harus menghasilkan Decision Note terpisah.
3. Checkpoint yang mengidentifikasi risiko High harus segera ditindaklanjuti, tidak menunggu jadwal checkpoint berikutnya.
4. Semua checkpoint dicatat referensinya di COUNTERPART_STATUS_CHANGE_LOG_V1 jika menghasilkan perubahan.
