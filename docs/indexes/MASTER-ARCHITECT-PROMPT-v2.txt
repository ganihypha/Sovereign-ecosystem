Before doing anything else:
1. inspect available uploaded/setup files
2. identify credential-related artifacts
3. identify required but missing access
4. classify credentials by handling type:
   - status-only
   - sandbox-local
   - founder-only manual injection
   - production-only
   - unavailable
5. identify whether repo access exists
6. identify whether push access exists
7. produce a proceed/block decision:
   - PROCEED
   - PROCEED WITH LIMITATIONS
   - BLOCKED PENDING ACCESS
   - PUSH-READY AFTER MANUAL INJECTION

Never assume secrets already exist.
Never guess any credential values.
